## Architecture & Key Decisions
- Multi-page architecture uses separate HTML files with full-page loads and standard anchor navigation — no SPA routing, no hash routing. Matches zero-build philosophy.
- Nav + scratchpad code intentionally duplicated between `index.html` and `news.html`. Accepted tradeoff: no shared includes, no build step. Future edits must touch both files.
- Weather widget stays homepage-only. Not duplicated to nav or news page.
- `.page-wrap` wrapper div introduced to allow nav to be a direct body child (full-width by default) without negative margins. Body padding moved entirely to `.page-wrap`.
- Active page state is hardcoded per file (not JS-detected). Consistent with per-page duplication approach.
- `#scratchpad-widget` positioning changed from `position: absolute` (inside `header`) to `position: relative` (inside `nav`). Panel positioning (`position: absolute; top: calc(100% + 6px)`) unchanged — still relative to the widget div.
- Nav `z-index: 100` (sticky) creates a stacking context. Scratchpad panel's `z-index: 150` is now local to that context. Net effect: modals correctly render above scratchpad panel (improvement from previous behavior where panel could obscure modals). Weather panel at root `z-index: 150` still renders above nav correctly.
