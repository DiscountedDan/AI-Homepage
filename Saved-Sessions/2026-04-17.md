## Architecture & Key Decisions
- CommonJS (module.exports) used for api/weather.js, matching the api/hello.js pattern
- Native fetch used in the serverless function (Node 18+ / Vercel default runtime) —
  no node-fetch package needed
- OPENWEATHERMAP_API_KEY read exclusively from process.env — never in code or HTML
- Forecast filtered server-side: 40 OWM 3-hour intervals grouped by date, today skipped,
  next 3 days returned with high/low computed from all intervals per day
- Noon icon (12:00:00) preferred per forecast day; falls back to middle-of-day entry
- ai_weather_city localStorage key: plain city name string, absent = use geolocation
- Widget placement mirrors scratchpad: position absolute, top-right of header
- Once-per-load data fetch only — no auto-refresh
