## Architecture & Key Decisions
- Gear icon is always visible (not hover-only) per spec, placed directly left of the checkbox in `.card-actions` flex wrapper
- Settings dropdown is a click-triggered absolute-positioned menu; closes on outside click via a single global document listener
- Three new modals follow the exact same HTML/CSS/JS pattern as existing v2 modals (static HTML, show/hide via `.hidden` class)
- Edit Card reuses the same "Add New..." dropdown logic as Add Resource, duplicated rather than abstracted (one-time use)
- Edit Review is a separate modal (not the existing review modal) to avoid tangling the "mark complete" flow with the "edit review" flow
- Editing a completed card's category only updates stored data — card stays in Completed section, does not revert to open
- Delete removes from both `ai_resources` and `ai_completion_data`; all writes go to localStorage before `renderAll()` so data is never at risk
- `closeAllModals()` and `closeAllSettingsMenus()` added as shared utilities; one modal at a time enforced everywhere
- No new localStorage keys or schema changes; no migration needed
