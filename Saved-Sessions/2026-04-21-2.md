## Architecture & Key Decisions
- `rss-parser@^3.13.0` chosen over hand-rolled XML parsing — first npm dependency in the project; `package.json` introduced at repo root
- `Promise.allSettled` used (contrast with `Promise.all` in `api/weather.js`) — multi-source aggregation must be fault-tolerant; one failing feed must not break the response
- Google AI short-string filter: if `contentSnippet` ≤ 30 chars for the `googleai` feed, treat summary as empty — avoids category labels ("Generative AI") rendering as article summaries
- Backend owns all summary sanitization (HTML strip, entity decode, truncation); frontend receives clean strings and only checks for truthiness before rendering
- Grouped-by-source layout within Lab Announcements: OpenAI block always renders first, Google AI block second — guarantees both labs are represented regardless of publishing cadence
- Response contract: `items` key is always present (never omitted) on per-feed error; simplifies frontend — `items.length` check is always safe

## Blockers & Risks

- Google AI summary omission is working as intended (titles + dates only for Google Research items), but worth keeping an eye on if the feed structure changes.
