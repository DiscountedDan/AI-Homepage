## Architecture & Key Decisions
- Scratchpad widget uses `position: absolute` inside a `position: relative` header ‚Äî zero impact on existing centered layout, no restructuring needed
- Scratchpad expanded state is a floating panel (z-index 150) anchored below the collapsed toggle, not a full-screen modal overlay
- Scratchpad uses `contenteditable` div storing innerHTML to `ai_scratchpad` ‚Äî enables rich text formatting without external libraries; trade-off is HTML tags visible in raw localStorage
- Font sizing uses `document.execCommand('fontSize')` with 7 discrete steps mapped to clean rem values via CSS `font[size]` selectors ‚Äî chosen over heading levels (h1/h2/h3) for continuous feel
- `mousedown` with `preventDefault()` added to all toolbar buttons to preserve contenteditable selection when clicking toolbar
- Per-card notes keyed by resource `id` (not title/index) to survive card edits; cleaned up on card delete
- Two new additive localStorage keys: `ai_scratchpad` (string HTML), `ai_card_notes` (object keyed by resource id)

## Blockers & Risks
- `document.execCommand` is technically deprecated in the web spec but remains fully supported in all browsers; no replacement exists for inline contenteditable formatting without a library
- Font size detection via `document.queryCommandValue('fontSize')` returns empty string when no explicit size has been set ‚Äî handled with `|| 3` fallback but worth monitoring
